# Implementation Plan: HelioMax Platform Transformation

**Branch**: `002-heliomax-platform` | **Date**: 2026-06-04 | **Spec**: [spec.md](./spec.md)

---

## Summary

Transform the HelioMax internal app from a passive HVAC sales record-keeper into an active AI-driven operations platform for 10 users. Work is structured as 7 sequential phases (0–6), each gated by a passing acceptance test. Phases 0–1 are unblocking infrastructure; Phase 2 (pipeline) is the data foundation for all AI/reporting phases.

---

## Technical Context

| Field | Value |
|---|---|
| **Language/Version** | TypeScript / Next.js 14.2 App Router |
| **Primary Dependencies** | Ant Design 5, Tailwind CSS, Recharts, @react-pdf/renderer, @anthropic-ai/sdk, Resend |
| **Storage** | Supabase (Postgres + RLS + Auth + Realtime) |
| **Testing** | Manual smoke test per phase (no automated test suite) |
| **Target Platform** | Web desktop, RTL Arabic, deployed on Vercel Pro |
| **Project Type** | Web application (brownfield — existing repo) |
| **Performance Goals** | Page load < 2s; session stable 8+ hrs; PDF export < 5s |
| **Constraints** | npm install --legacy-peer-deps; force-dynamic on live-data pages; passwords never touch AI |
| **Scale/Scope** | 10 internal users, ~500 leads/month, 98-model price list |
